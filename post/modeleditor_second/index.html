第一步. 功能思考

1. 功能描述

用于模型的编辑与制作。
左边：
    模型节点的层次结构列表，右键点击后弹出菜单：新建（长方体(支持梯形)、实心圆柱体、空心圆柱体，圆锥、凌锥、球体），
        复制，粘贴，改名，副本，展开，收拢，删除

右边：
    分2个视图
    文件视图：
        显示所有模型文件列表，底部有"筛选"按钮，点击该按钮弹出模型显示筛选界面（支持显示不同类别的模型以及排序）。

    属性视图：
        显示选中节点的详细信息
        位置 - 节点相对于父节点的位置偏移
        缩放 - 相对于父节点的缩放
        旋转 - 相对于父节点的旋转
        三角面 - 显示模型三角面数量
        子节点 - 显示子节点数量，子节点三角面数量
        轴心 - 模型逻辑中心点（x、y、z轴的取值都是0-1）
        长方体：
            简单设置 - 长、宽
            详细设置 - 前高，前宽，后高，后宽，左高，左宽，右高，右宽（四个面的高度、宽度都可以单独设置）
            贴图 - 左面贴图，右面贴图，上面贴图，下面贴图，前面贴图，后面贴图

        实心圆柱体：
            朝向 - x方向、y方向、z方向
            半径 - 圆柱两端的圆半径
            长度 - 圆柱体的长度
            精度 - 绘制精度
            贴图 - 两端贴图，柱体贴图

        空心圆柱体：
            朝向 - x方向、y方向、z方向
            内半径 - 空心圆的半径
            外半径 - 外圈半径
            长度 - 圆柱体长度
            内精度 - 内圆绘制精度
            外精度 - 外圆绘制精度
            两端贴图，柱体贴图

        圆锥：
            底面半径 - 底面半径
            高度 - 高度
            精度 - 绘制精度
            贴图 - 底面贴图、侧面贴图          

        凌锥：
            底面边数 - 如果为N，则表示底面为一个正N边形
            贴图 - 底面贴图，侧面贴图
            高度 - 高度

        球体：
            半径 - 球的半径
            贴图 - 贴面贴图
            精度 - 球的绘制精度

顶部：
    分两个视图
    模型视图：
        模型名字，以及作者
    动画视图：
        动画播放，以及帧列表

底部：
    模型总节点数
    模型总三角面数
    重置摄像机视角
    保存、新建、删除、另存、打开模型文件
    界面上控件tips显示


PS:
1.  一开始就需要按照命令模式开发，不能图快，以免后面改动太麻烦。
2.  每个子节点上需要有transform组件，便于计算。

2. 为什么要做这个功能？

3. 这个功能在整个游戏中处于什么样的位置？

4. 这能给游戏带来什么变化？

5. 与其他功能有何关联？

6. 开发优先级多高？

7. 开发代价多大？

8. 是否考虑过后续扩展？

9. 设计是否简洁明了，或者说是否太复杂？



第二步. UI原型制作

1. 是不是想好再动手？

2. 界面易用性高吗？

3. 是否使用了黄金分割？

4. 界面配色是否符合规范？

5. 字体大小是否符合规范？

6. 是否考虑界面动画？

7. 控件显示、宽高是否考虑多语言情况？

8. 代码是否执行编码规范？

9. 是否符合设计期望？



第三步. 逻辑功能实现

1. 逻辑文件是否拆分、组织妥当？

2. 代码是否执行编码规范？

3. 代码设计上是否考虑功能的扩展性？

4. 有没有考虑可能会造成性能瓶颈？

5. 有没有想过可能会有更简单的实现方式？



第四步. 评估完成情况

1. 界面表现是否符合设计期望？

2. 功能逻辑是否符合设计期望？

3. 是否留有瑕疵，是否有bug尚未修复？

4. 回头反思设计，是否有未考虑周到的地方？

5. 回头反思设计，是否有更好的表达方式？

6. 是否打包真机体验？
